  2. Adding a "Do Not Disturb" Mode

    We all know that people need to concentrate to work efficiently. This is why macOS and then all the operating systems have introduced the "Do not Disturb" mode, which allows to focus on your work.
    And I think we can do even better while integrating it to Mattermost.
    What is in the "Do not disturb" functionality ? It stops alls notification except the ones which doesn't disturb you.
    But the question is : why is it interesting to add it to Mattermost ?
    In a Mattermost server (for me), there are lots of different channels. Even the basics channels of Mattermost show us this : a "Town Square" which is the reception of your server, and an "Off Topic", which is the channel for entertainment. But in some server (as mine), there are lots of different "Off Topic" channels, for lots of different entertaining things.
    Here is why a do not disturb could be good.
    When you press this button, you won't receive any notification from all the channel that are not work channels, but in addition to that you will not have access to those disturbing channels.
    For example, if I have to work hard to finish a mission, I will enable the "Do not Disturb" mode in Mattermost, and I won't be able to see my notification for the disturbing channels, but I won't be able to access to those. Or maybe if I click on one of those channels, I will get an alert which will say me "Hey, you are on "Do Not Disturb" mode, you have to work (or something like that)
    What do you think about that ?

  15. Support searching for Recent Mentions for multiple mention strings

    When enabling additional terms which trigger mentions in the notifications setting of the application, the search for recent mentions looks only for messages that contain all of the mention trigger words.

    Notifications are properly triggered for each of the mention trigger words, but when you go to search for recent mentions, it fails to show most of your recent mentions as they typically don't include both/all mention trigger words.

    As an example, I have both a team mention "myteam" and mentions for my @username. When I click "Recent Mentions" the search bar contains:

    @username myteam

    Per documentation on https://docs.mattermost.com/help/getting-started/searching.html, the intention is for search to return results with all searched terms contained in the search bar in a single message.

    It would be useful for the recent mentions search to be able to use "OR" or "ANY" logic for each of the words in the search bar instead of "ALL" logic.

    In my particular use case, I have several different mention triggers, so searching for each term individually is not a reasonable solution, and doesn't provide the benefit of the interleaved timeline of the various different mention terms.

  17. Emoji Delete Audit Logging

    As of now, there is no audit logged in to the Audits table when an emoji gets deleted. The only way to find out about this is through the mattermost.log. For example:

    {"level":"debug","ts":XXXXXXXX,"caller":"web/handlers.go:101","msg":"Received HTTP request","method":"DELETE","url":"/api/v4/emoji/<emojiid>","requestid":"XXXXXXXX","status_code":"200"}

    It will be better to have this information logged into the Audits table for System Administrators to have the ability to trace this information.
